# SEO Page Strategy Skill

**Status:** Draft  
**Owner:** GovCon Giants / Growth + Product  
**Purpose:** Turn public MI data into useful search pages that attract qualified GovCon users while gating the full workflow inside Market Intelligence.

## When To Use This

Use this skill when planning or reviewing public pages meant to rank in Google and convert search visitors into MI Free, MI Pro, or white-glove conversations.

Trigger examples:
- "How is SEO coming along?"
- "Build contractor sales pages like HigherGov or GovTribe."
- "What public data should we show and what should be gated?"
- "This page should attract users from Google."
- "We need .org to .com redirect planning."

## Core Thesis

Public SEO pages should help a visitor answer one useful question quickly, then show that MI has the deeper workflow.

Examples:
- "How much federal revenue has this contractor won year over year?"
- "Which agencies buy this category?"
- "Which recompetes are coming up?"
- "Which companies are winning in my NAICS?"
- "What opportunities match my business?"

## Page Types

| Page Type | Public Value | Gated MI Value |
| --- | --- | --- |
| Contractor profile | Basic company, agency, NAICS, sales trend | Full history, contacts, competitors, pipeline actions |
| NAICS market page | Market size and top agencies | Saved searches, alerts, forecast/recompete feed |
| Agency opportunity page | Buying patterns and upcoming needs | Matching, pain points, partner suggestions |
| Recompete page | Expiring contract summary | Watchlist, incumbent analysis, proposal plan |
| Forecast page | Upcoming procurement preview | Full forecast search, tracking, team handoff |

## Public/Gated Rule

Show enough to be genuinely useful. Gate the workflow that creates ongoing advantage.

### Public

- Page title and plain-English summary
- Basic profile fields
- Aggregate sales trend
- Top agencies or NAICS
- A few sample contracts or opportunities
- Last updated date
- Clear source notes

### Gated

- Full table exports
- Saved searches
- Contact details
- Teaming CRM actions
- Proposal and pipeline actions
- Full recompete and forecast workflow
- Personalized alerts and briefings

## SEO Requirements

Every public page should include:

- Canonical `.com` URL when the page is public marketing/SEO
- `mi.govcongiants.com` URL only when the page is an app/product surface
- Unique title and meta description
- Structured data where useful
- Indexable summary copy
- Internal links to related contractor, agency, NAICS, forecast, and recompete pages
- A clear CTA to MI Free or MI Pro
- A data freshness note

## Conversion Requirements

Each page should answer:

1. What did the visitor come here to learn?
2. What should they do next?
3. What does MI unlock that this public page does not?
4. Is the right next step MI Free, MI Pro, or white-glove?

## Review Output

When using this skill, produce:

- Search intent
- Target query examples
- Public data to show
- Gated data to protect
- CTA and next step
- Internal links
- Canonical URL
- Data source notes
- Implementation risks

## Definition Of Done

A page strategy is ready when it can:

- Rank for a real search query
- Give the visitor useful public value
- Avoid exposing paid/customer data
- Move qualified users into MI
- Support the larger promise: help small businesses find and win federal contracts
