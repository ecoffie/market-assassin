## Implementation Plan (Real Public Data - No AI Generation)

### Data Sources to Scrape/Parse

| Source | URL | Data Type | Agencies Covered |
|--------|-----|-----------|------------------|
| **GAO High Risk List** | gao.gov/high-risk-list | Problem areas | All agencies |
| **OIG Semiannual Reports** | oversight.gov | Top challenges | All agencies with OIG |
| **Agency Strategic Plans** | Agency websites | Priorities | All agencies |
| **Congressional Justifications** | congress.gov/budget | Budget priorities | All agencies |
| **Federal IT Dashboard** | itdashboard.gov | IT spending/projects | All agencies |
| **FPDS Contract Patterns** | SAM.gov/USASpending | What they buy | All agencies |
| **NDAA Provisions** | congress.gov | Legislative mandates | Affected agencies |

### Phase 1: Expand Hardcoded Data (NOW)
Add more agencies to existing TypeScript files:
1. `IG_CHALLENGES_BY_AGENCY` in `federal-oversight-data.ts` (currently 20 agencies → expand to 100+)
2. `AGENCY_SPENDING_PRIORITIES` in `federal-oversight-data.ts` (currently 10 agencies → expand to 50+)
3. `GAO_HIGH_RISK_AREAS` - already comprehensive

### Phase 2: Build OIG Report Scraper
Create a scraper that:
1. Fetches OIG Top Management Challenges from oversight.gov
2. Parses PDF/HTML reports
3. Extracts challenge categories
4. Stores in JSON format

### Phase 3: Build Strategic Plan Scraper
Create a scraper that:
1. Fetches agency strategic plans
2. Extracts goals and objectives
3. Maps to pain points format

### Phase 4: Automated Pipeline
Build an admin endpoint that:
1. Pulls latest data from all sources
2. Merges with existing pain points
3. Validates data quality
4. Updates `agency-pain-points.json`

### Admin Endpoint

```bash
# Preview expansion
GET /api/admin/expand-pain-points?password=xxx&mode=preview

# Execute expansion
POST /api/admin/expand-pain-points?password=xxx&mode=execute

# Add single agency
POST /api/admin/expand-pain-points?password=xxx&mode=single&agency=NAVFAC
```
