## B) AI TECHNICAL CONTENT (for Servexo's submission)

### Q1 — LLM/RAG production capability (≤500 char survey box + the long version for the cap statement)

**SURVEY (≤500 char):**
> Production LLM/RAG grounded in authoritative data: documents chunked w/ structured metadata,
> embedded to a vector store, retrieved with metadata filtering, and answered with mandatory inline
> **citations to source + as-of date**. Hallucination reduced via retrieval-only grounding,
> confidence thresholds, and abstain-on-low-evidence. Continuous eval vs a labeled gold set.
> Security: least-privilege, encryption, audit logging. Built for sustainment, not a demo.

**LONG (capabilities statement):**
- **Architecture:** ingestion → normalization (NUL-safe, legacy-system connectors) → semantic
  chunking with structured metadata (agency, NAICS/PSC, date, doc type) → embeddings → vector store →
  metadata-filtered retrieval → LLM answer with enforced citation + as-of date.
- **Grounding/citations:** every figure traces to its source record and date; answers that can't be
  grounded abstain rather than fabricate (our core "grounded, not generic AI" discipline).
- **Hallucination reduction:** retrieval-only context, citation enforcement, confidence gating,
  abstain-on-insufficient-evidence, LLM-as-judge fact-check on high-stakes output.
- **Model eval:** labeled gold dataset; precision/recall on retrieval + factuality scoring; regression
  on each change.
- **Security/sustainment:** least-privilege service accounts, encryption in transit/at rest, full
  audit logging, provider-agnostic model fallback chain (no single-vendor lock-in), cost controls;
  designed to run continuously, not a POC.

### Q2 — AI workflow automation + agentic controls (the key discriminator)

**SURVEY (≤500 char):**
> Match the tool to the task: rules-based for deterministic steps, AI-assist for human-in-the-loop
> drafting, retrieval-enabled for grounded answers, agentic only where multi-step autonomy earns its
> risk. Controls: human approval routing, **bounded permissions, rollback, full audit trail,
> traceability**; safeguards vs unauthorized actions, invalid tool calls, prompt injection, and data
> leakage (input/output filtering, allow-listed tools, least privilege).

**LONG:** the decision rubric (rules → AI-assist → retrieval → agentic, escalating by autonomy/risk);
every agentic action runs under explicit approval gates, scoped permissions, reversible operations,
and an audit log; defenses are concrete — tool allow-listing, injection filtering, output DLP, and
least-privilege so an agent can't act outside its lane. (This mirrors how Mindy's proposal/agentic
features already enforce grounding + bounded action.)
